# Domain Docs

This repo uses a single-context domain layout.

## Before exploring, read these

- `CONTEXT.md` at the repo root
- ADRs under `docs/adr/`

If these files are absent, proceed silently. Producer skills can create or update them when project language or decisions need to be captured.

## Use the glossary's vocabulary

When output names a domain concept in an issue, refactor proposal, test name, or implementation note, use the term as defined in `CONTEXT.md`.

## Flag ADR conflicts

If output contradicts an existing ADR, surface it explicitly rather than silently overriding it.
